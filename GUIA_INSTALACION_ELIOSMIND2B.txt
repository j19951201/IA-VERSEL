Proyecto EliosMind 2B es un Launcher profesional de Windows para ejecutar y
monitorear el cerebro GGUF (modelo de lenguaje) utilizado en IA Juris.

La aplicación:
✓ Monitorea la conectividad del cerebro GGUF
✓ Verifica que las APIs estén funcionales
✓ Valida los tokens de autenticación
✓ Ejecuta automáticamente el stack de IA
✓ Genera logs de monitoreo en tiempo real
✓ Se instala como una aplicación de Windows estándar

DESPUÉS DE INSTALAR:

1. INICIAR DESDE EL ESCRITORIO:
   • Haz doble clic en "Proyecto EliosMind 2B.lnk"

2. O DESDE EL MENÚ INICIO:
   • Busca "Proyecto EliosMind 2B"
   • Haz clic en "Proyecto EliosMind 2B"

3. LA APLICACIÓN HARÁ:
   • Iniciará automáticamente el stack de IA
   • Lanzará el servidor de desarrollo (Vercel dev en puerto 3000)
   • Comenzará a monitorear cada 90 segundos:
     ✓ Estado del cerebro GGUF
     ✓ Validez del token de autenticación
     ✓ Conectividad de la API
   • Generará un archivo de log con los resultados

4. ARCHIVO DE LOG:
   Los logs se guardan en: C:\Users\[Tu Usuario]\Desktop\proyecto eliosmind2b.monitor.log
   
   Ejemplo de log:
   [2026-05-07 13:27:21] GLOBAL=OK | Cerebro GGUF /health=OK | API key/token=OK | App conectada=OK

⚙ CONFIGURACIÓN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Para PERSONALIZAR la aplicación, edita estos archivos:

1. C:\Program Files\EliosMind2B\proyecto_eliosmind2b.config.json
   
   Parámetros configurables:
   • ProxyBaseUrl        → URL del proxy GGUF (default: http://127.0.0.1:11436)
   • ProxyToken          → Token de autenticación (default: corvinus-hard-token)
   • Model               → Modelo a usar (default: corvinus-unico:latest)
   • IntervalSeconds     → Intervalo de monitoreo en segundos (default: 90)
   • AutoStartStack      → Iniciar stack automáticamente (default: true)
   • AutoStartAppDev     → Iniciar app dev automáticamente (default: true)

2. C:\Program Files\EliosMind2B\.env.local
   
   Variables de entorno:
   • INTERNAL_AI_API_URL → URL de la API del cerebro
   • INTERNAL_AI_API_KEY → Clave de API
   • INTERNAL_AI_AUTH_HEADER → Header de autenticación
   • INTERNAL_AI_AUTH_SCHEME → Esquema de auth (Bearer, etc)

3. C:\Program Files\EliosMind2B\.env.development.local
   
   Variables para entorno de desarrollo

🐛 SOLUCIÓN DE PROBLEMAS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PROBLEMA: "Acceso denegado" durante la instalación
───────────────────────────────────────────────────
SOLUCIÓN:
• Ejecuta el instalador como administrador
• Haz clic derecho → "Ejecutar como administrador"

PROBLEMA: El cerebro GGUF no responde
─────────────────────────────────────
SOLUCIÓN:
• Verifica que Ollama esté corriendo
• Ejecuta: ollama serve
• Verifica el puerto 11434

PROBLEMA: El monitor muestra "FAIL" en conectividad
───────────────────────────────────────────────────
SOLUCIÓN:
• Verifica el archivo .env.local con las credenciales correctas
• Valida el token de autenticación en proyecto_eliosmind2b.config.json
• Comprueba que el proxy esté escuchando en puerto 11436

PROBLEMA: Vercel dev no inicia
──────────────────────────────
SOLUCIÓN:
• Asegúrate de que Node.js está instalado
• Verifica que el puerto 3000 está disponible
• En terminal: netstat -ano | findstr :3000
• Si está en uso: taskkill /PID [PID] /F

PROBLEMA: Puerto ocupado
───────────────────────
SOLUCIÓN:
• Encuentra qué proceso usa el puerto: netstat -ano | findstr :[PUERTO]
• Termina el proceso: taskkill /PID [PID] /F
• O reinicia tu computadora
